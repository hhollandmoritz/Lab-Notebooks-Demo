Jupyter Notebooks¶

This is a jupyter notebook, in this notebook I will demonstrate some of the basics of using jupyter notebooks.

Cell Types¶

Jupyter notebooks have two main types of cells

  1. markdown cells (like this one)
  2. coding cells (like the one below)
In [2]:
# by default, coding cells are in python
print("I am a coding cell")

I am a coding cell

  • Markdown cells can be used for creating headings, or for writing commentary about the code, such as explanations or version numbers.

  • Coding cells can be used for analysis.

By default the coding language is python and code is run in-line, in the notebook. But you can use bash script (terminal commands) or R in ipython as well.

Running Terminal Commands in Jupyter Notebooks¶

If you're just running one command, you can use an exclamation for quick commands.

Note: This won't work for cd, to change directories you need to use the "magic" cells (cells with %%). However, it is better practice to just use full paths.

In [3]:
!pwd

/home/hannah/Documents/Fierer_lab/Lab Meeting

To make a bash script, you'll need to use "cell magic" like below.

In [4]:
%%bash
for i in a b c;
do
echo $i
done

a
b
c

Using Jupyter Notebooks Remotely¶

Let's say your data is on a remote server (i.e. Microbe) and you want to interact with it through a jupyter notebook. To set this up, follow the steps below:

This tutorial was adapted from this website: https://amber-md.github.io/pytraj/latest/tutorials/remote_jupyter_notebook

Pre-requisites - make sure jupyter notebook is installed on your own computer and on the server you are trying to connect with.

Part 1: On the remote server

  1. Open a terminal window and log onto the remote server

    ssh username@microbe.colorado.edu

  2. cd into the directory with your notebook (or the directory where you want to create a new notebook)

  3. type jupyter notebook --no-browser --port=1234

    Explanation:

    --no-browser

    This command tells jupyter notebook not to try to launch a browser window, which is good since the default "browser" in a terminal window, is pretty horrible.

    --port=1234

    This command tells jupyter notebook that instead of opening a browser it should send the notebook to a special outlet called a port. Ports are specified by numbers, the numbers can be almost anything. It is probably a good idea to make sure that multiple users do not use the same port numbers since conflicts can arrise if two people request the same port at the same time.

Part 2: On your local computer

  1. Open a second terminal window. On your local computer type ssh -N -f -L localhost:8888:localhost:1234 username@your_remote_host_name

    Explanation:

    -N

    This command works with the -f command to help the ssh command run in the background.

    -f

    This command tells the terminal to put ssh command in the background just before it runs. This is mainly so that after you run the command, you can still use the terminal that you ran it in.

    -L

    This command specifies the port-to-port connections. It is followed by a localhost:8888:localhost:1234 you can read this as instructions to the computer. "Connect port 8888 on the current host to port 1234 on the remote host."

  2. Type in your server password when prompted.

  3. Open a web browser (chrome, firefox, etc.) and type localhost:8888 in the address line.

Don't forget

You're running the notebook on the server so if you choose to run R in your notebook, you'll need to install all of the R packages on the server before you do that.
